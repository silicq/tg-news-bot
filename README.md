# tg-news-bot — автопостинг новостей в Telegram на Cloudflare

Автономный бот на **Cloudflare Workers**, который по расписанию (Cron Triggers)
забирает новости из RSS, отбирает самые интересные под тему канала с помощью
**Workers AI**, генерирует к ним картинку, пишет короткую оригинальную подпись и
публикует пост в Telegram-канал. Всё рассчитано на **бесплатный тариф**
(Workers Free + Workers AI 10 000 нейронов/день).

## Что он делает за один запуск

```
cron → fetchFeeds() → parseItems() → filterAlreadyPosted (D1)
     → rankInteresting (Workers AI, 1 батч-вызов на всех кандидатов)
     → для топ-N:
          → makeCaption()      (Workers AI: короткий оригинальный текст)
          → makeImagePrompt()  (Workers AI: визуальный промпт из заголовка)
          → generateImage()    (Workers AI: байты картинки)  ⟍ фолбэк: og:image
          → postToTelegram()   (sendPhoto: фото + подпись + ссылка)
          → recordPosted (D1)
     → обновить расход нейронов (D1)
```

Один упавший источник или один неудачный пост **не роняют** весь запуск.
Повторный запуск **не публикует дубликаты** (дедупликация по sha-256 в D1).

## Стек

- **Cloudflare Workers** — рантайм, TypeScript, сборка через Wrangler.
- **Cron Triggers** — запуск по расписанию.
- **D1 (SQLite)** — история опубликованного + счётчик нейронов за сутки.
- **Workers AI** — текстовая модель (отбор + подпись + промпт картинки) и
  image-модель (генерация картинки).
- Источники новостей — **только RSS/Atom**, без платных API.

## Быстрый старт

### 0. Предварительно

- Аккаунт Cloudflare (бесплатный) и установленный Node.js 18+.
- Установить зависимости:

  ```bash
  npm install
  ```

- Авторизоваться в Cloudflare (для локального деплоя):

  ```bash
  npx wrangler login
  ```

### 1. Создать Telegram-бота и канал

1. В Telegram напишите **@BotFather** → `/newbot` → получите **токен** вида
   `123456:ABC-DEF...`.
2. Создайте канал (или возьмите существующий).
3. Добавьте бота в канал **администратором** с правом «Публикация сообщений».
4. Узнайте `chat_id` канала:
   - публичный канал → используйте `@username`;
   - приватный → перешлите любой пост канала боту **@getidsbot** (или подобному)
     и возьмите id вида `-1001234567890`.

### 2. Создать базу D1 и применить миграцию

```bash
# создать базу (один раз)
npx wrangler d1 create news_bot
```

Команда напечатает `database_id`. Вставьте его в `wrangler.toml`:

```toml
[[d1_databases]]
binding = "DB"
database_name = "news_bot"
database_id = "СЮДА_ВАШ_ID"
migrations_dir = "migrations"
```

Примените миграцию (создаёт таблицы `posted` и `budget`):

```bash
# для локального wrangler dev
npm run migrate:local
# для продакшена (удалённая D1)
npm run migrate:remote
```

> Миграцию на удалённую базу достаточно применить **один раз**. Если меняете
> схему — добавляйте новый файл `migrations/0002_*.sql` и применяете снова.

### 3. Задать секреты и переменные

**Секреты** (НЕ коммитятся в git). Локально — файл `.dev.vars`
(скопируйте из `.dev.vars.example`). В проде — дашборд или CLI:

```bash
npx wrangler secret put TELEGRAM_BOT_TOKEN
# id админа в Telegram — только он может выполнять команды бота (/test, /stats):
npx wrangler secret put ADMIN_ID
# защита ручного триггера /run, /setup и секрет вебхука Telegram:
npx wrangler secret put MANUAL_TRIGGER_TOKEN
```

> Свой `ADMIN_ID` (числовой id) узнайте у бота **@userinfobot**.

**Несекретные переменные** уже лежат в `wrangler.toml` (`[vars]`).
Отредактируйте под себя как минимум:

- `TELEGRAM_CHANNEL_ID` — `@username` или числовой id канала;
- `RSS_FEEDS` — JSON-массив URL источников;
- `CHANNEL_THEME` — описание темы/аудитории (по нему идёт отбор).

Полный список переменных — в разделе [Конфигурация](#конфигурация).

### 4. Локальный тест

```bash
npm run dev          # поднимет wrangler dev на http://localhost:8787
```

В отдельном окне дёрните ручной триггер (нужен `MANUAL_TRIGGER_TOKEN` в `.dev.vars`):

```bash
curl "http://localhost:8787/run?token=ВАШ_MANUAL_TRIGGER_TOKEN"
```

Проверка живости без побочных эффектов: `curl http://localhost:8787/health`.

Логи (что нашёл, что отобрал, сколько нейронов потратил) видны в консоли
`wrangler dev`. Можно также эмулировать cron-хендлер — запустите dev с флагом
`--test-scheduled` и дёрните служебный путь:

```bash
npx wrangler dev --test-scheduled
curl "http://localhost:8787/__scheduled?cron=0+*/3+*+*+*"
```

### 5. Деплой вручную

```bash
npm run deploy       # = wrangler deploy
```

После деплоя задайте секреты и (при необходимости) vars **в дашборде**
Cloudflare: *Workers & Pages → ваш Worker → Settings → Variables and Secrets*.

## Деплой из GitHub (Cloudflare Workers Builds, CI/CD)

Цель: пуш в `main` → автоматическая сборка и деплой.

1. **Создайте репозиторий на GitHub** и запушьте проект:

   ```bash
   git init
   git add .
   git commit -m "Initial commit: tg-news-bot"
   git branch -M main
   git remote add origin https://github.com/USER/REPO.git
   git push -u origin main
   ```

   > Секреты не попадут в git — `.dev.vars` и `.wrangler/` в `.gitignore`.
   > В репозитории только несекретные `vars` из `wrangler.toml` — это нормально.

2. В дашборде Cloudflare: **Workers & Pages → Create → Workers →
   Connect to Git** (Workers Builds). Выберите GitHub-репозиторий.

3. Настройте билд:
   - **Build command:** `npm install`
   - **Deploy command:** `npx wrangler deploy`
   - **Branch:** `main` (deploy on push).

4. **Секреты и vars задайте в дашборде** (Settings → Variables and Secrets):
   `TELEGRAM_BOT_TOKEN`, при желании `MANUAL_TRIGGER_TOKEN`. Vars можно оставить
   в `wrangler.toml` либо переопределить в дашборде.

5. **Миграция D1** не выполняется автоматически билдом. Примените её один раз
   локально (`npm run migrate:remote`) **или** добавьте отдельный шаг в Deploy
   command:

   ```
   npx wrangler d1 migrations apply news_bot --remote && npx wrangler deploy
   ```

6. Проверьте: сделайте пустой коммит и `git push` — в разделе **Builds** Worker'а
   должна стартовать сборка и деплой.

## Команды бота и самопроверка

Бот принимает команды от **админа** в личке. Команды обрабатываются через
вебхук Telegram, который указывает на сам Worker. Реагирует только пользователь
с id `ADMIN_ID` — остальные игнорируются.

### Подключить вебхук (один раз после деплоя)

Откройте в браузере (или `curl`) служебный путь — он зарегистрирует вебхук
автоматически:

```
https://tg-news-bot.<ваш-сабдомен>.workers.dev/setup?token=ВАШ_MANUAL_TRIGGER_TOKEN
```

В ответ придёт `{"ok":true,...}`. `MANUAL_TRIGGER_TOKEN` используется и как пароль
к `/setup`, и как `secret_token` вебхука (Telegram присылает его в заголовке —
так чужие POST'ы на `/telegram` отсекаются).

> Альтернатива вручную:
> `curl "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://.../telegram&secret_token=<MANUAL_TRIGGER_TOKEN>&allowed_updates=[\"message\"]"`

### Команды

| Команда | Что делает |
|---|---|
| `/test` | **Самопроверка.** Берёт свежую новость, прогоняет весь конвейер (подпись + генерация картинки) и присылает готовый пост вам в личку. Затем проверяет связь с каналом (доступ + право бота на публикацию). По итогу — сводка; при любой ошибке приходит уведомление с её текстом. |
| `/stats` | Статистика за текущие сутки (UTC): сколько постов **опубликовано**, сколько новостей **пропущено**, расход нейронов и остаток до лимитов Cloudflare (фритир 10 000 и внутренний бюджет). |
| `/run` | Запустить обычный цикл публикации немедленно. |
| `/help` | Список команд. |

Счётчики берутся из D1 (таблица `budget`, одна строка на UTC-сутки) — **отдельная
миграция не нужна**: Worker сам создаёт таблицы и добавляет колонку `skipped_count`
при старте (см. `src/schema.ts`). «Пропущено» = новости, которые бот рассмотрел,
но отклонил как не подходящие по теме (ниже `MIN_SCORE`) или пропустил при
`NO_IMAGE_BEHAVIOR=skip`. Счётчик нейронов — **оценка** (биндинг Workers AI не
отдаёт реальный расход), сброс в 00:00 UTC.

## Конфигурация

Всё задаётся через `wrangler.toml` (`[vars]`) и секреты.

| Переменная | Тип | Назначение | Дефолт |
|---|---|---|---|
| `TELEGRAM_BOT_TOKEN` | **secret** | Токен бота от @BotFather | — |
| `ADMIN_ID` | **secret** | Telegram id админа (только он шлёт команды `/test`, `/stats`, `/run`) | — |
| `MANUAL_TRIGGER_TOKEN` | secret | Пароль к `GET /run` и `/setup` + секрет вебхука. Пусто → триггер и команды выключены | — |
| `TELEGRAM_CHANNEL_ID` | var | `@username` или числовой id канала | `@your_channel` |
| `RSS_FEEDS` | var | JSON-массив URL фидов | набор научпоп/позитив |
| `CHANNEL_THEME` | var | Описание темы/аудитории для отбора | позитив/наука/космос |
| `CAPTION_LANG` | var | Язык подписи | `ru` |
| `CAPTION_TONE` | var | Тон подписи | живой, без кликбейта |
| `SOURCE_LABEL` | var | Текст ссылки на источник | `Источник` |
| `MAX_AGE_HOURS` | var | Брать записи не старше N часов | `24` |
| `MAX_POSTS_PER_RUN` | var | Лимит постов за один запуск | `2` |
| `MAX_POSTS_PER_DAY` | var | Лимит постов за сутки | `4` |
| `MIN_SCORE` | var | Мин. AI-оценка (0–100) для публикации | `55` |
| `IMAGE_MODE` | var | `generate` или `og_first` | `generate` |
| `OG_FALLBACK_NEURON_THRESHOLD` | var | Ниже этого остатка нейронов — берём og:image | `2000` |
| `NO_IMAGE_BEHAVIOR` | var | Нет картинки совсем: `og` / `text` / `skip` | `og` |
| `DAILY_NEURON_BUDGET` | var | Дневной бюджет нейронов (из 10 000) | `9000` |
| `TEXT_MODEL` | var | ID текстовой модели Workers AI | `@cf/meta/llama-3.1-8b-instruct-fast` |
| `IMAGE_MODEL` | var | ID image-модели Workers AI | `@cf/black-forest-labs/flux-1-schnell` |
| `IMAGE_STEPS` | var | Шагов диффузии (flux: 1–8) | `4` |
| `EST_NEURONS_*` | var | Оценки стоимости шагов (см. ниже) | 60/30/15/150 |
| `HISTORY_RETENTION_DAYS` | var | Через сколько дней чистить историю в D1 | `30` |

### Режимы картинки

- **`generate`** (по умолчанию): генерируем картинку через Workers AI. Если
  дневной бюджет на исходе (`OG_FALLBACK_NEURON_THRESHOLD`) **или** генерация
  упала — пытаемся вытащить `og:image` со страницы статьи (это бесплатно по
  нейронам). Если og нет — действуем по `NO_IMAGE_BEHAVIOR`.
- **`og_first`**: сначала пытаемся `og:image` (бесплатно), генерацию используем
  только если og нет. Сильно растягивает фритир и позволяет много постов.

### `NO_IMAGE_BEHAVIOR` (когда картинки нет вообще)

- `og` (дефолт) — публикуем как текстовое сообщение (картинка была желательна,
  но не обязательна);
- `text` — публикуем текстом (`sendMessage` + ссылка);
- `skip` — пропускаем пост.

## Расписание (Cron)

В `wrangler.toml`:

```toml
[triggers]
crons = ["0 */3 * * *"]   # каждые 3 часа = 8 запусков/сутки
```

Альтернативы: `"0 */4 * * *"` (6/сутки), `"0 6,12,18 * * *"` (3/сутки).
Чем чаще запуск, тем больше суммарный расход нейронов за сутки — см. ниже.
Время в cron — **UTC**.

## Прикидка расхода нейронов

> Важно: Workers AI **не возвращает** реальный расход нейронов в ответе биндинга.
> Поэтому бюджет считается по **оценкам** (`EST_NEURONS_*`), специально слегка
> завышенным — чтобы защита срабатывала **заранее**, а не после превышения.
> Реальные числа ниже посчитаны по тарифам Cloudflare (июнь 2026).

Тарифы (нейронов на 1М токенов): `llama-3.1-8b-instruct-fast` ≈ 4 119 вход /
34 868 выход. `flux-1-schnell` ≈ 9,6 нейрона за шаг диффузии.

**На 1 пост (режим `generate`, flux 4 шага):**

| Шаг | Оценка кода (`EST_*`) | Реально (≈) |
|---|---|---|
| Подпись (текст) | 30 | ~6 |
| Промпт картинки (текст) | 15 | ~2 |
| Генерация картинки (4 шага) | 150 | ~40 |
| **Итого на пост** | **~195** | **~48** |

**Общий на запуск (шарится между постами):** отбор/ранжирование — оценка 60,
реально ~19 нейронов.

**За сутки при дефолте (8 запусков, 4 поста/сутки):**

| | Оценка кода | Реально (≈) |
|---|---|---|
| Ранжирование 8× | 480 | ~150 |
| 4 поста | 780 | ~190 |
| **Итого/сутки** | **~1 260** | **~340** |

То есть даже по завышенным оценкам — ~1 260 из 9 000 бюджетных (и из 10 000
фритира). Реально — ~340 нейронов/сутки, **~3–4 % фритира**. Запас огромный:
можно спокойно увеличить `MAX_POSTS_PER_DAY`.

В режиме **`og_first`** генерация картинок почти не используется (og:image
бесплатны), поэтому расход падает до ~25 нейронов/пост — постов можно делать
кратно больше.

**Как защита не даёт пробить 10 000/сутки:**
перед каждым дорогим вызовом `BudgetTracker.canAfford()` проверяет
`использовано_сегодня + стоимость_шага ≤ DAILY_NEURON_BUDGET (9000)`. Счётчик
живёт в таблице `budget` (одна строка на UTC-сутки, сброс в 00:00 UTC). Когда
остаток падает ниже `OG_FALLBACK_NEURON_THRESHOLD`, генерация заменяется на
бесплатный og:image. Плюс жёсткие лимиты `MAX_POSTS_PER_RUN` и
`MAX_POSTS_PER_DAY`.

## Как обеспечивается отсутствие дублей

- Ключ записи — `sha-256(guid || link)`.
- Перед отбором кандидаты фильтруются по таблице `posted`.
- После успешной публикации запись добавляется (`INSERT OR IGNORE`).
- Любой повторный/параллельный запуск просто не найдёт новых кандидатов.

## Заметки по копирайту и Telegram

- Подпись пишется **своими словами** (модель инструктирована не копировать
  предложения из статьи), 2–4 предложения.
- Подпись обрезается под лимит Telegram **1024 символа**, ссылка на источник —
  кликабельная (`parse_mode=HTML`): `🔗 <a href="...">Источник</a>`.
- `sendPhoto` отправляет картинку как `multipart/form-data` (Blob из Worker),
  либо как URL-строку (для og:image — Telegram сам её скачивает).

## Траблшутинг

- **Бот не постит:** проверьте, что бот — админ канала с правом публикации, что
  `TELEGRAM_CHANNEL_ID` верный, и что миграция D1 применена.
- **`Telegram sendPhoto failed`:** для og-фолбэка картинка по URL должна быть
  публично доступна и < 5 МБ; иначе пост уйдёт на следующий запуск.
- **Мало/нет кандидатов:** уменьшите `MIN_SCORE` или увеличьте `MAX_AGE_HOURS`,
  проверьте, что фиды в `RSS_FEEDS` отдают свежие записи.
- **Проверить актуальные ID моделей/цены:** см. документацию Cloudflare Workers
  AI (модели и нейроны периодически меняются) — `TEXT_MODEL`/`IMAGE_MODEL`
  вынесены в конфиг именно для лёгкой замены.
